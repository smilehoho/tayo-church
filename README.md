# 따요 처치 출석 체크

> 팔찌에 붙은 **QR을 폰으로 찍기만** 하면 출석이 기록되는, **설치가 필요 없는** 웹앱입니다.
> **인터넷이 끊겨도** 동작하고, 연결되면 **구글 시트**에 자동으로 모입니다.

**▶ 바로 열어보기: https://smilehoho.github.io/tayo-church/**

---

## 어떤 고민에서 시작했나

교회에서 출석을 체크해 보신 분이라면 이런 순간이 익숙하실 거예요.

- 종이 명부에 이름을 하나하나 부르며 체크하다 보면 **시간이 오래 걸립니다.**
- 사람이 많거나 아이들이 몰려오면 **누가 왔는지 빠뜨리기 쉽습니다.**
- 나중에 "이번 달에 몇 번 나왔지?", "요즘 안 보이는 사람은 누구지?"를 세려면 **집계가 번거롭습니다.**
- 손으로 적은 걸 엑셀로 다시 옮기다 보면 **실수도 생깁니다.**

게다가 필리핀 현지처럼 **인터넷이 불안정한 환경**에서는, "인터넷이 있어야만 되는 앱"은 예배 시간에 멈춰버릴까 봐 마음 놓고 쓰기 어렵습니다.

이 프로젝트는 그 고민을 **가장 단순한 방법으로** 풀어보려는 시도입니다.

---

## 어떻게 해결했나

한 문장으로 말하면 이렇습니다.

> **성도마다 QR을 하나씩 나눠주고, 교사가 폰 카메라로 찍기만 하면 출석이 저장됩니다.**

실제 흐름은 다섯 걸음이면 끝납니다.

1. **번호 발급** — 성도마다 고유 번호를 정합니다. (예: `0142`)
2. **QR 배포** — 그 번호를 QR로 만들어 **팔찌·목걸이·출석카드**에 붙여 나눠줍니다.
3. **스캔** — 예배 입구에서 교사가 폰으로 앱을 열고 QR을 비춥니다.
4. **자동 저장** — 화면에 **"출석 완료 ✓"**. 그걸로 끝입니다.
5. **확인** — 담당자는 구글 시트에서 오늘 누가 왔는지 한눈에 봅니다.

팔찌를 안 가져온 사람은 **번호를 손으로 입력**해도 됩니다.

---

## 왜 이렇게 만들었나 (설계에 담긴 이유)

교회 현장에서 실제로 쓸 수 있으려면, 편리함보다 **"어떤 상황에서도 멈추지 않는 것"**이 더 중요하다고 생각했습니다.

| 현장의 걱정 | 이 앱의 답 |
|---|---|
| 앱 설치가 번거롭다 | **설치가 없습니다.** 주소만 열면 바로 씁니다. (원하면 홈 화면에 아이콘으로 추가 가능) |
| 인터넷이 자주 끊긴다 | **끊겨도 됩니다.** 오프라인에서도 스캔·저장되고, 연결되면 자동으로 서버에 올라갑니다. |
| 같은 사람을 두 번 찍으면? | **걱정 없습니다.** 같은 날 같은 번호는 자동으로 한 건으로 처리됩니다. |
| 장비를 새로 사야 하나? | **아니요.** 지금 쓰는 스마트폰이면 충분합니다. |
| 비용이 드나? | **무료입니다.** 무료 웹 호스팅과 구글 시트만 씁니다. |
| 개인정보가 걱정된다 | **이름 없이 번호만** 다룹니다. (아래 "개인정보와 보안" 참고) |

## 실제로 써보기

폰 브라우저(크롬·사파리)로 **https://smilehoho.github.io/tayo-church/** 를 여세요.

1. **스캔 시작** 버튼을 누르고 카메라를 허용합니다.
2. 팔찌의 QR을 화면에 비추면 → **"출석 완료 ✓"** 소리·진동과 함께 저장됩니다.
3. 팔찌가 없으면 **번호로 체크** 칸에 번호를 입력합니다.
4. **오늘 출석 / 미동기화** 숫자와 **최근 스캔** 목록으로 상태를 확인합니다.
5. 자주 쓴다면 **홈 화면에 추가**해 앱처럼 사용하세요.
   - 아이폰(사파리): 공유 → *홈 화면에 추가*
   - 안드로이드(크롬): 메뉴 → *앱 설치 / 홈 화면에 추가*

> **QR로 미리 연습해 보기:** 아무 QR 생성기에서 `0142` 또는 `MEMBER-0142` 같은 글자로 QR을 만들어 찍어보세요. 앱은 **글자 끝의 숫자만** 성도 번호로 알아봅니다. (`MEMBER-0142` → `0142`)

## 개인정보와 보안

교회의 소중한 정보를 다루기에, 처음부터 조심스럽게 설계했습니다.

- **이름·연락처 없이 "번호"만 저장합니다.** 번호와 실제 이름을 잇는 명부는 교회가 **따로, 안전하게** 보관하시면 됩니다. 앱이나 시트가 새어 나가도 그것만으로는 누구인지 알 수 없습니다.
- **비밀 토큰으로 보호합니다.** 앱이 구글 시트로 출석을 보낼 때 정해진 비밀 값을 함께 보내, **외부인이 가짜 출석을 밀어 넣지 못하도록** 서버가 막습니다.
- **항상 암호화된 연결(HTTPS)** 로만 오갑니다.

## 어떻게 만들어졌나 (조금 더 자세히)

기술이 궁금하신 분을 위한 요약입니다.

- **설치형 웹앱(PWA):** 특별한 서버 프로그램 없이, 웹 파일만으로 동작합니다.
- **카메라 QR 인식:** `html5-qrcode` 라이브러리를 앱 안에 함께 담아(외부 의존 없음) 사용합니다.
- **오프라인 저장:** 스캔 기록을 폰 브라우저 안(IndexedDB)에 저장하고, 서비스워커가 앱 자체를 저장해 **인터넷 없이도 열립니다.**
- **집계 서버:** 무료인 **구글 시트 + Apps Script**. 같은 날·같은 번호·같은 기기는 **서버에서도 한 번 더 중복을 걸러**, 오프라인에서 밀렸던 기록이 다시 전송돼도 안전합니다.
- **호스팅:** **GitHub Pages** (무료, 자동 HTTPS). 그래서 폰에서 카메라와 오프라인 기능이 모두 정상 동작합니다.

## 직접 운영·설치하려면

교회에서 이 앱을 자기 것으로 만들어 쓰려는 담당자를 위한 안내입니다.

### 1) 내 컴퓨터에서 먼저 실행해 보기

```bash
cd attendance-pwa
python3 -m http.server 8000
```

브라우저에서 **http://localhost:8000** 을 엽니다. (`localhost`는 보안 예외라 카메라·오프라인이 그대로 동작합니다.)

### 2) 구글 시트에 연결하기

1. 구글 시트를 새로 만들고 **확장 프로그램 → Apps Script** 를 엽니다.
2. `apps-script/Code.gs` 내용을 붙여넣고 **웹 앱으로 배포**합니다. (실행: 나 / 액세스: 모든 사용자)
3. 배포 후 나온 URL을 `config.js` 의 `ENDPOINT` 에 넣습니다.
4. **비밀 토큰 설정:** `config.js` 의 `TOKEN` 과 **똑같은 값**을 Apps Script의 *프로젝트 설정 → 스크립트 속성* 에 `SHARED_TOKEN` 이라는 이름으로 넣고 **재배포**합니다.
   - 이 속성이 있으면, 토큰이 다른 요청은 서버가 거부합니다.
   - 참고: 토큰은 앱에도 담기므로 자동 스캔봇 차단용이며, 완벽한 비밀은 아닙니다.

### 3) 인터넷에 공개하기 (GitHub Pages)

이 폴더(`attendance-pwa/`)를 저장소의 **루트**로 올린 뒤, 저장소의 **Settings → Pages** 에서 배포 소스를 **브랜치 `main` / 폴더 `/ (root)`** 로 지정하고 저장하면 됩니다.

- 주소는 `https://<아이디>.github.io/<저장소이름>/` 형태로 자동 생성됩니다.
- `.nojekyll` 파일이 함께 있어야 `vendor/` 등이 빠짐없이 게시됩니다. (이미 포함되어 있습니다)
- 파일을 수정하면 `sw.js` 맨 위의 캐시 버전(`tayo-att-vN`)을 올려야 방문자들이 새 파일을 받습니다.

## 폴더 안에 무엇이 들어 있나

```
attendance-pwa/            ← 이 폴더 전체가 하나의 앱입니다
├─ index.html              화면 (버튼·목록 등 눈에 보이는 부분)
├─ app.js                  스캔 · 저장 · 동기화를 담당하는 두뇌
├─ sw.js                   서비스워커 (인터넷 없이도 열리게 하는 장치)
├─ manifest.webmanifest    앱 정보 (이름·아이콘·색)
├─ icon.svg                앱 아이콘
├─ config.js               설정 (구글 시트 주소 · 비밀 토큰)
├─ .nojekyll               GitHub Pages가 파일을 손대지 않게 하는 표시
├─ vendor/                 QR 인식 라이브러리 (앱 안에 함께 보관)
└─ apps-script/Code.gs     구글 시트에 붙여넣는 서버 코드
```

---

이 앱은 필리핀 **따요 처치(Tayo Church)** 를 위해 만들어졌지만, 번호를 정하고 QR을 나눠주는 방식은 **어느 교회에서든** 그대로 쓸 수 있습니다. 출석 체크에 쏟던 시간을, 다시 사람에게 쓰시길 바랍니다.
